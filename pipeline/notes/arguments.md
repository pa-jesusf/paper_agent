## 第一章 绪论

**核心论点**: 恶意软件已从个体攻击演变为家族化、变种化的系统性威胁；勒索软件中普遍存在的"休眠"行为导致动态分析方法大面积失效；需要从行为语义层面结合静态分析重新构建识别体系。

**支撑论据**:
1. **统计数据驱动**: 2024年上半年全球47%金融机构遭勒索攻击 [statista]；60%+勒索样本具备休眠特征 [node4, blackfog, levelupmsp, cyberdivision]
2. **动态方法失效**: ShieldFS [continella2016shieldfs]、Redemption [kharraz2017redemption] 等动态检测方案无法捕获休眠期行为；CanCal [wang2024cancal] 报告 FNR>35%
3. **攻防极性不可消除**: 98.2%恶意样本同时展现攻击性与防御性行为极性，且O-D准正交(r=0.48)
4. **5项实证发现**: 437样本98家族实证研究→确立"动态解包+静态分析"框架

## 第二章 相关工作

### 2.1 恶意软件静态分析与行为语义提取

**核心论点**: 静态分析从简单特征（N-gram、PE头）发展到行为语义层面，但现有方法未能系统化定义和提取攻防行为语义。

**支撑论据**:
1. **经典静态方法**: N-gram [abou2004n]、PE头学习 [raff2017learning]、Drebin [arp2014drebin]
2. **加密检测**: 熵分析方法 [lyda2007using]；加壳检测 [li2023packgenome, aghakhani2020malware]
3. **Windows恶意软件分析**: API调用行为 [cheng2018towards, amer2020dynamic]
4. **语义鸿沟**: 混淆(控制流平坦化、不透明谓词、MBA)导致字面级特征失效，需要语义等价识别

### 2.2 基于机器学习的恶意软件识别

**核心论点**: ML方法从传统分类器到DNN再到GNN逐步演进，但缺乏对行为语义的直接建模和对时序漂移的鲁棒性。

**支撑论据**:
1. **传统ML**: 特征工程+分类器 [schultz2000data, masud2008scalable]；综述 [ucci2019survey]
2. **DNN方法**: 原始字节学习 [raff2017learning, catanzaro2018malware, krvcal2018deep]；对抗鲁棒性不足 [lucas2023adversarial, lucas2024training]
3. **GNN方法**: GraphSAGE [hamilton2017inductive] 为基础模型 → **Gap: 需要补充GNN4Malware相关工作**
4. **时序挑战**: 概念漂移 [chen2023continuous, dambra2023decoding] → 时序设置评估必要性

### 2.3 大语言模型在安全分析中的应用

**核心论点**: LLM为代码理解和语义分析带来突破性能力，但在恶意软件行为识别领域应用尚处于起步阶段。

**支撑论据**:
1. **LLM+安全**: 静态二进制污点分析 [liu2025llm]；LLM-as-a-Judge评估范式 [Gu2024ASO]
2. **本文贡献定位**: 三阶段LLM管线(SLEI/SEFEI/CAR)是首个系统化将LLM用于恶意软件攻防行为语义识别的框架
3. **⚠️ 文献缺口**: 需大量补充LLM4Security、LLM4RE、LLM4Vulnerability相关文献(估计10-15篇)

### 2.4 恶意软件家族分析

**核心论点**: 家族分析对理解恶意软件演化和提升识别精度至关重要，但现有研究缺乏基于攻防行为语义的系统化多粒度分析框架。

**支撑论据**:
1. **加密密钥分析**: [cicala2020analysis] 分析勒索软件家族加密密钥生成差异
2. **数据破坏实证**: [hou2024empirical] 勒索攻击中的数据破坏行为实证
3. **本文贡献定位**: PolarProfile → 34家族437样本 → 3个RQ → 6个Finding
4. **⚠️ 文献缺口**: 需大量补充Malware Phylogeny、Family Clustering、AVclass、代码相似度分析(估计8-12篇)

### 2.5 勒索软件识别技术专题

**核心论点**: 勒索软件检测从运行时行为监控发展到多模态融合，但静态先验知识驱动的方法仍有巨大空间。

**支撑论据**:
1. **动态方法**: ShieldFS [continella2016shieldfs]、UNVEIL [kharaz2016unveil]、RWGuard [mehnaz2018rwguard]、R-Trap [ganfure2023rtrap]
2. **混合方法**: RansomSpector [tang2020ransomspector]、McIntosh [mcintosh2021dynamic]
3. **静态方法**: Zero-Ran Sniff [cen2024zero]、元学习 [zhu2022few]、Ranker [zhang2024ranker]
4. **对抗性评估**: I/O模拟攻击 [zhou2023limits]

## 第三章 知识库构建

**核心论点**: 攻防行为语义需要分层定义和系统化提取，三层知识库 $\mathbb{K}=\{\mathbb{K}_{lex}, \mathbb{K}_{sem}, \mathbb{K}_{sta}\}$ 实现了从字面到语义到统计的完整覆盖。

**支撑论据**:
1. **实证基础**: 2914篇论文调研(6人-月)；712+204+57条规则
2. **词法层**: 712条规则覆盖API/常量/字符串，标注Kappa=0.80-0.85
3. **语义层**: 204条规则，13个开源加密库 [openssl...gnutls]，186个标注加密函数
4. **统计层**: 57维特征向量，KW检验53/57维显著(p<0.05)
5. **LLM三阶段识别**: SLEI/SEFEI/CAR，GPT-4o最优(F1=0.89/0.91/0.93)

## 第四章 家族分析

**核心论点**: 勒索软件家族在攻防行为上呈现高度结构化的可区分性，攻击性行为(O)与防御性行为(D)构成准正交的互补极性。

**支撑论据**:
1. **F1**: 7维行为画像维度效率高, η²=0.29-0.43全部大效应
2. **F2**: 攻防极性跨家族成立(98.2%样本), O-D准正交(r=0.48)
3. **F3**: 同一家族拓扑结构内聚(5/5 CV通过, 11/11 KW显著)
4. **F4**: O子图分散、D子图紧致；图拓扑η²=0.55-0.72为最强模态
5. **F5**: 三层差异化覆盖(Lex-Sem r=0.06不冗余)
6. **F6**: Statistical 53/54特征大效应(top η²=0.77)

## 第五章 PolarCatch

**核心论点**: 将攻防行为极性通过双GNN分离建模，结合统计特征三通道融合，可实现高精度且对抗鲁棒的恶意软件识别。

**支撑论据**:
1. **设计依据**: 第四章F2(极性独立) + F4(图拓扑最强) → 双图分离架构
2. **有效性**: Chronological F1=0.931, +20.9% vs ZRS [cen2024zero]
3. **鲁棒性**: 5类对抗攻击F1≥0.96 [xie2017mitigating, pappas2012smashing, koo2016juggling]
4. **消融**: 三通道缺任一则F1下降3-12个百分点
5. **效率**: 9.8秒/样本, $0.047/样本

**核心论点**: 三大贡献构成逻辑闭环——知识库→家族规律→识别模型，回答了绪论提出的三个核心问题。

**支撑论据**:
